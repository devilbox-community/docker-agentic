Ansible is used to generate Dockerfiles from Jinja2 templates.

## Quick start

```bash
# From the repository root:
make gen
```

This runs:
1. `bin/gen-agentic-tools.py` — scans `agentic_tools/` and `extra_tools/`, generates group_vars
2. Ansible playbook via `cytopia/ansible:2.12-tools` container — renders Dockerfiles

## Manual run

```bash
docker run --rm \
    -e USER=ansible \
    -e MY_UID=$(id -u) \
    -e MY_GID=$(id -g) \
    -v ${PWD}:/data \
    -w /data/.ansible \
    cytopia/ansible:2.12-tools ansible-playbook generate.yml \
        -e ansible_python_interpreter=/usr/bin/python3 \
        -e docker_user=devilboxcommunity \
        --forks 50 \
        --diff
```

## Templates

| Template | Generates |
|----------|-----------|
| `DOCKERFILES/Dockerfile-base.j2` | `Dockerfiles/base/Dockerfile` |
| `DOCKERFILES/Dockerfile-work.j2` | `Dockerfiles/work/Dockerfile` |
| `DOCKERFILES/Dockerfile-agentic.j2` | `Dockerfiles/agentic/Dockerfile-{tool}` (per-agent) |

## Inventory

One `[agentic]` group with 8 hosts:

```
base, work, claude-code, codex, copilot, opencode, pi-coding-agent, reasonix
```

Each host maps to a build stage. The `base` and `work` hosts render the
base/work Dockerfiles; agent tool hosts render per-agent Dockerfiles under
`Dockerfiles/agentic/`.

## Group vars

| File | Source | Purpose |
|------|--------|---------|
| `agentic.yml` | `agentic_tools/` | Per-agent harness CLI definitions |
| `work.yml` | `extra_tools/` | Shared spec/workflow tool definitions |
| `all-ansible.yml` | Manual | Ansible config, template paths, versions |

All `agentic.yml` and `work.yml` are auto-generated by `bin/gen-agentic-tools.py`.
Do not edit them manually.
